4. Module-by-Module Breakdown

🔐 4.1 Authentication & Role Management

  • JWT (access token + refresh token) – stored securely (httpOnly cookie or mobile secure storage).
  • Middlewares: authMiddleware (verifies token), roleMiddleware(allowedRoles).
  • 4 roles, 4 scopes:
RoleScope & Permissions
it_adminFull system: manage tenants, approve/block, impersonate any tenant dashboard, view all logs.
tenant_adminOwn tenant: manage branches, users, products, view all reports, subscription.
branch_managerSingle branch: sales, customers, debt, reports (branch only).
cashierSingle branch: create sales, add customers/products on‑fly, view debtors (no reports).

🧾 4.2 Subscription & Tenant Onboarding (Kiro System)

  1. Landing page (public) – shows pricing plans (Basic: 1 branch, Premium: unlimited branches).
  2. Registration form – business name, owner, email, phone, address, selected plan, number of branches (based on plan).
  3. New tenant inserted with status = 'pending'.
  4. IT Admin receives notification (in‑app + email).
  5. IT Admin reviews → clicks Approve → status becomes active, subscription start/end dates set.
  6. Tenant Admin receives login credentials, sets password, creates branches & users.
  7. Cron job (daily) checks expiry_date → if expired → status = suspended.
  8. Payment integration (planned): via eDahab, Zaad, or card – recorded in subscription_payments.

🛒 4.3 Sales Process – Smart & Fast (Zero friction)

UI/UX flow (App/Web):

  • Cashier opens New Sale screen.
  • Customer section:
    • Searchable dropdown with existing customers (by name/phone).
    • If not found → “+ Add new customer” opens quick modal (name, phone, optional) – saves instantly and selects.
  • Product section:
    • Search products by name or SKU.
    • If not exists → “+ Add new product” modal (name, price, stock, category) – adds and continues.
  • Payment selection: toggle between Paid (lacag bixiyey) iyo Credit (deyn).
  • If Credit → debt_amount = total - paid (paid may be partial). Customer debt_balance increased.
  • Invoice: auto‑generated, printable (thermal printer via Bluetooth).

Backend logic (transaction):

BEGIN;
  INSERT INTO sales (...);
  INSERT INTO sale_items (...);
  UPDATE products SET stock = stock - quantity WHERE id = ...;
  IF payment_status = 'credit' THEN
    UPDATE customers SET debt_balance = debt_balance + debt_amount WHERE id = ...;
  END IF;
COMMIT;

📉 4.4 Debt Management – Labo Tab

Tab 1: Deyn (Debtors)

  • Shows all customers with debt_balance > 0.
  • Search field: filter by name/phone.
  • Date filtering: by last sale date.
  • Click on customer → opens debt detail page:
    • List of credit sales (date, invoice, amount, paid, remaining).
    • Payment history.
    • Button “Record Payment” – enter amount, update debt_balance, insert into payments.

Tab 2: La Bixiyey (Paid Sales)

  • List of sales with payment_status = 'paid'.
  • Same search (customer) and date filters.

📊 4.5 Reports & Analytics

  • Sales report: total sales, paid, credit – daily, weekly, monthly, custom range.
  • Product report: best selling products, low stock alert.
  • Payment report: total cash received, credit given, collection efficiency.
  • Branch performance: compare revenue per branch (tenant admin).
  • Export: PDF, Excel (CSV).
  • Visibility: IT Admin → all tenants; Tenant Admin → own tenant; Branch Manager → own branch only.

🧑‍💻 4.6 IT Admin Dashboard – Tenant Management & Impersonation

  • List all tenants with status badges (pending, active, suspended, blocked).
  • Quick actions: Approve, Suspend, Block, Activate, Extend subscription.
  • Click on tenant row → “View Dashboard” (impersonation).
    • IT Admin views the exact dashboard that the tenant admin sees (sales, products, reports, branches).
    • Clearly labelled “You are viewing as [business name]”.
    • Can switch back to IT panel.
  • View tenant’s users, branches, subscription expiry.

8. Deployment & CI/CD

8.1 Backend (Node.js + MySQL)

  • Process: PM2 or Docker (Node).
  • Database: Managed MySQL (AWS RDS, DigitalOcean).
  • Reverse proxy: Nginx + SSL (Let's Encrypt).

8.2 Frontend (Flutter)

  • Web: flutter build web → deploy to Vercel/Netlify/cPanel.
  • Mobile: Build APK / IPA → distribute via private link or MDM.

9. Security & Performance

🔒 Security

  • Helmet.js – secure headers (HSTS, XSS filter).
  • Rate limiting – express-rate-limit (auth, sales).
  • Input validation – express-validator.
  • SQL injection – prevented by Sequelize (parameterized).
  • XSS – escape output, sanitize HTML.
  • CORS – restrict to trusted domains.

⚡ Performance

  • Redis – session store, cache frequent queries (products, customers).
  • Database indexes – on foreign keys, search fields (name, phone, sku).
  • Pagination – all list endpoints (limit/offset).
  • Read replicas – for heavy report queries (future).

10. Additional Recommendations

  • ✅ Audit Trail – every INSERT/UPDATE/DELETE logged in audit_logs.
  • ✅ Automated Backups – daily MySQL dump → S3 or local storage.
  • ✅ Monitoring & Error Tracking – Sentry for backend, Crashlytics for Flutter, Logtail for logs.
  • ✅ Scalability – horizontal scaling with load balancer (sticky sessions off).
  • ✅ Multi-language – implement i18n (Somali / English) using Flutter intl.
  • ✅ Offline support – queue sales when offline, sync when back online (Hive + connectivity).
  • ✅ Receipt printing – ESC/POS Bluetooth for mobile, PDF for web.
